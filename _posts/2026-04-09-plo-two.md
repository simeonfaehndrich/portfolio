> Identify information needs and respond through the design and provision of information products and services

For my Python course I worked with two other students to develop a tool that could be used by museums to digitize their accessioning workflow. A teammate had experience with the Qualicum Beach Museum so we imagined them as a test case—he created a hundred hypothetical items the museum might have.

The tool our team created worked by prompting the user to input metadata elements of an item, which were saved to a text file. This file worked as a database, from which different details could be outputted.

My role in the project was developing the functions to aggregate data by year of the item. I created a script that output the number of items for each decade by mining the database for segments of the string at the appropriate field. This would be useful for a museum to give a quick overview of the historical quality of their collection, especially with the loss of institutional knowledge with high turnover. Potential use cases include curating an exhibit based on a decade or investing in developing a museum’s collection on timeframes that are weak in material.

The second functionality I contributed was similar to the previous but aimed at locating item donors. This is important for museums because knowing who bestows the most could help with future fundraising efforts or getting a sense of the origin and background of the collection. While this was a very close problem to the first I had solved, I took a different approach. In the first, I had hard coded a search for each decade such as “189*” in the “Date” field, but this had limitations such as not accounting for dates far in the future or past (that I didn’t account for). So for my second go at this kind of problem, I looked at the libraries folks had contributed to the open source internet and used that as a base. The result was much lighter weight in terms of lines then the decade search and produced excellent search results.

```python
def donorContribution(totalRecord):
    donor = Counter() #Assigns the Counter function to a variable
    for item in totalRecord: #Loops through each item the list of dictionaries
        donor[item["Donor Name"]] += 1 #If the value associated with the "Donor Name" key is the same, the count goes up for that value
    
    for key, value in donor.most_common():
        print("{}: {}".format(key, value)) #Formats the donor list so each donor has their own line, as well as removing them from the dictionary
```
